- Use Docker Hub to learn VOLUME path and versions needed to run it
URL: https://hub.docker.com/layers/library/postgres/9.6.1/images/sha256-4023a747a01a29f94939c41fc5c2441837d9ebc05030c2681dc47853d73978c5
* docker image pull postgres:9.6.1
* docker image inspect postgres:9.6.1 -> search for Volumes in JSON
    * /var/lib/postgresql/data

- Create custom volume
docker volume create psql-data

- Create a postgres container with named volume psql-data using version 9.6.1
docker container run -d --name psqldb -e POSTGRES_PASSWORD=password -v psql-data:/var/lib/postgres/data postgres:9.6.1

- Check voluems 
docker volume ls

- Check logs
docker container logs -f psqldb

- Stop container
docker container stop psqldb

- Create a new postgres container with same named volume using 9.6.2
docker container run -d --name psqldb2 -e POSTGRES_PASSWORD=password -v psql-data:/var/lib/postgres/data postgres:9.6.2

- Check logs to validate
docker container logs -f psqldb
